## 🧠 Behavioral Rules

- ✅ Only show articles published within the last **24 hours**
- ✅ Saved articles **persist across page refreshes** (localStorage → Supabase later)
- ✅ On refresh: show saved articles **plus** any newly scraped articles
- ✅ If no new articles exist, do not error — show last known state
- ✅ Scraper runs on a **24-hour cron / schedule**
- ✅ Design must be **beautiful, interactive, and visually premium**
- ❌ Never show articles older than 24 hours in the main feed
- ❌ Never hardcode credentials (`.env` only)
- ❌ No Supabase integration until Phase 2 is explicitly started

---

## 🏛️ Architectural Invariants

1. LLMs are probabilistic — all business logic lives in deterministic Python tools (`tools/`).
2. `.env` stores all secrets. Never hardcode credentials.
3. `.tmp/` is ephemeral — never treat it as a data store.
4. `data/articles.json` is the local source of truth (pre-Supabase).
5. If logic changes → update the SOP in `architecture/` **first**, then update the code.
6. A project is only "Complete" when the final Payload reaches its cloud destination.
